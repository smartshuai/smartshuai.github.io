Gradle在Android Studio的目录结构及配置内容总结

发表于 2016-12-02   |     |   阅读次数

本文主要对Android Studio中使用的Gradle目录结构，以及各文件配置内容进行描述。

Project层

  1. 根目录下的settings.gradle 主要作用是告诉Gradle在构建app的过程中它需要包含哪些moudles.一般情况下include ‘：app’就可以了。
  2. 根目录下的build.gradle文件定义了适用于所有modules的配置信息。默认情况下，它包含了buildscript {}语句块来定义Gradle仓库和一些依赖。

Moudle层

  1. 该层文件build.gradle作用是为所在的module配置构建信息，在这个文件中可以自定义打包选项等。此外，该文件中的设置会覆盖project层build.gradle文件以及main/ app manifest文件中的设置信息。
    该文件中有几项比较重要，其中
    • android {}
      compileSdkVersion指定了Gradle构建我们的项目时所要使用的Android API level。
      buildToolsVersion 指定了Gradle构建app时使用的SDK build tools、命令行以及编译器的版本。
    • defaultConfig {}
      applicationId 定义了app的完整包名，当我们的应用发布到应用市场时，这个包名就相当于应用的id。
      minSdkVersion app运行时所需要的最小API level，即app所兼容的Android最低版本号。
      targetSdkVersion 定义测试app使用的API level。
    • buildTypes {}
      其中可以分别定义debug and release两个apk版本的配置信息，对应于/projectName/app/build/outputs/apk目录下的两个Android studio生成apk版本。
    • dependencies {}
      主要用于添加依赖的各种库。
  2. local.properties中最重要的是定义了SDK的安装位置，但是这个文件是Android Studio自动生成的，所以当我们修改SDK路径的的时候应该通过Android Studio中的设置选项修改，而不是直接修改这个文件。

Source Sets层

主要用来组织源代码和资源文件

  1. AndroidManifest.xml 在该文件中添加源码所需的权限，添加自定义的Activity、BroadCastReceiver、Service、ContentProvider等。
